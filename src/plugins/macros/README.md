# macros

TypeScriptToLua plugin that applies compile-time macros: a build header, file metadata tokens, and `LuaTable`
cast helpers.

The `$` helpers are imported from the `xray16/macros` module (which ships a real runtime, so the same code runs
under jest/node without mocks or globals). This plugin strips that import at build time and folds each usage; the
`$` tokens are recognized by name whether imported or ambient.

Every feature is on by default. The token-driven features are inert unless the matching `$` token appears in the
source, so leaving them enabled is safe.

```json
{ "name": "xray16/plugins/macros", "buildTimestamp": true, "fileName": true, "dirName": true, "castHelpers": true }
```

## Configuration

### `buildTimestamp`

Prepends a generation comment to every emitted Lua file. Defaults to `true`.

```lua
-- Generated by xrf util at: <build time>

local ____exports = {}
```

### `fileName` / `dirName`

Replaces the `$filename` and `$dirname` globals with the compile-time file and directory name. Both default to
`true`. Lua has no convenient runtime API for this, so resolving it at build time is simpler.

`$filename` becomes the file name without extension. `$dirname` becomes the containing directory name. Only these
exact identifiers are replaced; similarly named locals are left alone.

```typescript
declare const $filename: string;
declare const $dirname: string;

// In scripts/quests.ts:
export const meta = [$filename, $dirname]; // -> {"quests", "scripts"}
```

### `castHelpers`

Expands `LuaTable` interop helpers. Defaults to `true`.

- `$fromObject`, `$fromArray`, `$fromLuaArray`, `$fromLuaTable` are unwrapped to their single argument and removed
  from the runtime. They exist for typing only.
- `$isNil` and `$isNotNil` compile to `== nil` and `‾= nil`.

```typescript
declare function $fromObject<T>(value: unknown): T;
declare function $isNil(value: unknown): boolean;
declare function $isNotNil(value: unknown): boolean;

export function run(value: unknown): unknown {
  const casted = $fromObject<number>(value);

  return [$isNil(casted), $isNotNil(casted), casted];
}

// casted        -> value
// $isNil(x)     -> x == nil
// $isNotNil(x)  -> x ‾= nil
```

A cast helper called with anything other than one argument produces a build error: `Invalid transformer call,
expected function to have exactly 1 argument.` The first argument is still unwrapped so a single mistake does not
cascade.

## Limitations

- The `$` tokens are not valid Lua identifiers. Disabling `fileName`, `dirName`, or `castHelpers` while the matching
  token is still used leaves the token in the tree, and TSTL then reports it as an invalid ambient identifier. Keep a
  feature enabled while its token is in use; the toggles exist to turn a feature off once its tokens are gone.
- `buildTimestamp` uses the build machine clock, so emitted output is not reproducible byte-for-byte across builds.
